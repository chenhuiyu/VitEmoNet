# VitEmoNet

Transformer-based EEG emotion recognition on the SEED dataset.

## Overview

VitEmoNet is a deep learning project for 3-class emotion recognition from EEG DE features.
The repository focuses on a clean training pipeline, reproducible experiments, and practical extensibility.

## Features

- Transformer backbone for temporal-channel EEG representation learning
- Support for preprocessed SEED DE features (`input_data_1d` / `input_data`)
- Weighted training for class imbalance handling
- Validation/test evaluation and training logs
- Inference entrypoint for downstream usage

## Repository Structure

```text
VitEmoNet/
├── train.py              # training entrypoint
├── inference.py          # inference script
├── transformer.py        # Transformer model definition
├── vivit.py              # alternative video-style transformer module
├── callbacks.py          # custom callbacks and checkpoint logic
├── metrics.py            # confusion matrix and metric utilities
├── data.py               # data processing helpers
├── data_1d.py            # 1D feature generation script
├── data_2d.py            # 2D feature generation script
└── logger.py             # logging utility
```

## Dataset

This project is designed for **SEED** EEG emotion data (3 classes).

Expected preprocessed files:

- `input_data_1d/train_data.npy`
- `input_data_1d/train_label.npy`
- `input_data_1d/val_data.npy`
- `input_data_1d/val_label.npy`
- `input_data_1d/test_data.npy`
- `input_data_1d/test_label.npy`

(Or the same file names under `input_data/` depending on feature format.)

## Environment

Recommended:

- Python 3.8+
- TensorFlow 2.x
- NumPy
- scikit-learn
- Matplotlib

Install dependencies (example):

```bash
pip install tensorflow numpy scikit-learn matplotlib scipy
```

## Training

```bash
python3 train.py
```

Training artifacts are saved under:

- `save/<timestamp>/train.log`
- `save/<timestamp>/model_summary.log`
- checkpoints/plots generated by callbacks

## Inference

```bash
python3 inference.py
```

Adjust model/data paths in the script based on your environment.

## Model

Default model in `train.py`:

- input shape: `(5, 25, 62)`
- stacked Transformer encoder blocks
- temporal/channel pooling + MLP head
- 3-class softmax output

## Customization

You can modify in `train.py`:

- learning rate
- batch size
- epochs
- transformer depth / head size / MLP units
